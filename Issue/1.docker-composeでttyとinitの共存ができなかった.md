## Issue

docker-compose.ymlについて

```
version: '3.7'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - '8080:8080'
    tty: true
    init: true
    volumes:
      - ./:/app/
```

のように設定した際に`docker-compose up --build`にて立ち上げたコンテナが即終了となってしまった

## Answer
原因は不明だが、
```
tty: true
init: true
```
の共存を止めるとコンテナが落ちなかった
これだとPID1問題を解決できないのでは？とも思うので分かり次第このIssueに追加する
